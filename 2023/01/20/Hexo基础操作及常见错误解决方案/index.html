Hexo基础操作及常见错误解决方案
2023-01-20 安河藏山

⚪文件目录

public：目录下存放的是我们生成的「静态页面」
source/_posts：目录下存放的是我们写的「文章」；
themes：目录下存放的是「博客」的主题；
_config.yml：是「博客全局配置」文件；
_config.landscape.yml：是「博客主题配置」文件

⚪Hexo 常用命令

hexo n "postName"  // 新建文章
hexo clean  // 清除缓存
hexo g  // 生成静态网页
hexo s http://localhost:4000/  // 本地预览
hexo d  // 部署到Github

⚪推送到Github

在VsCode中打开blog文件夹，找到_config.yml文件，找到deploy，按照以下格式进行修改：

deploy:
  type: git
  repo: git@github.com:你的用户名/你的用户名.github.io.git
  branch: master

deploy:
  type: git
  repo: git@Sol-B:Sol-B/Sol-B.github.io.git  // 如果有两个推送库Sol-B是库的别称
  branch: master

❌错误01

hexo d命令报错ERROR Deployer not found: git

这是因为没安装 hexo-deployer-git 插件，在站点目录下输入下面的插件安装就好了：

npm install hexo-deployer-git --save

❌错误02

解决hexo文章推送后自定义域名失效

在 source 下创建名为 CNAME 文件，不带文件后缀，并自定义域名 （www.XXXXX.com） 写入其中，注意前面要加www。
